# Kodak Gold 200 LUT vs 真实胶片 — 色彩差异研究报告

**日期**: 2026-06-14
**方法**: 定量 LUT 分析 + 社区参考 + 色彩科学原理
**结论**: 存在 **5 个具体差异** 和 **1 个结构性缺口**

---

## 1. 执行摘要

当前 gold200 LUT 成功捕捉了 Kodak Gold 200 的**整体暖金色调方向**，但在多个色域维度上存在偏差：蓝色过暖过饱和丢失、绿色缺乏黄绿偏移、红色的「标志性橙移」仅作用于肤色而未触及纯红色、影调曲线中 per-channel contrast 的方向与胶片物理结构相反。此外流水线存在固有的结构性限制（无负片反转、无层间串扰、Rec.709 色域限制）。

---

## 2. 维度分析

### 2.1 影调曲线

| 参数 | 当前值 | 分析 | 建议 |
|------|--------|------|------|
| contrast | 1.02 | 接近中性，消费级胶片「pop」感可能不足 | 考虑 1.05-1.08 |
| per_channel_contrast | [1.02, 1.00, **0.96**] | ⚠️ **B 通道最低与胶片物理相反** | 考虑 [1.00, 1.00, 1.04] 或至少 B ≥ G |
| black_lift | 0.0020 | 合理 — 模拟片基密度 | 保持 |
| highlight_shoulder | start=0.72, power=1.20 | 合理 — 柔和高光 roll-off | 保持 |

**证据**: 彩色负片的三层乳剂结构中，蓝色感光层位于最上层（离镜头最近），通常具有最高的 γ 值。5219 LUT 的 B 通道 per_channel_contrast 为 1.06（最高），符合此模型。

**根源**: 当前 per_channel_contrast 的分配似乎是为了「创造暖调效果」（R 高 → 暖），而非基于胶片物理建模。

### 2.2 红色渲染 — 「标志性橙移」的不完整实现

**发现**: orange_push=10° 存在 **hue wraparound 问题**。

<details>
<summary>技术细节</summary>

在 `engine/core.py:330-332`:
```python
red_mask = (h >= 0.0) & (h <= 40.0)
if orange_push != 0.0 and np.any(red_mask):
    h[red_mask] = h[red_mask] + orange_push
```

当纯红色 (R=1,G=0,B=0) 转换到 HSL 后，H 值可能为 360°（而非 0°）。360° 不在 `[0, 40]` 范围内，导致 orange_push **对饱和红色完全无效**。

**定量验证**: 纯红 (1,0,0) 经过 LUT → 输出 H=360°（无偏移）；肤色 (0.9,0.7,0.5) → 输出 H=39.1°（+9.1° 偏移）。
</details>

**影响**: 预设声称的「Gold 200 签名：红色→琥珀」实际上变成了「肤色→琥珀」，对纯红色毫无效果。

**建议**: 修改 red_mask 包含 360° wraparound 或调整 hue 计算逻辑。

### 2.3 蓝色渲染 — 过度暖化与参数矛盾

| 参数 | 当前值 | 效果 |
|------|--------|------|
| teal_push | -5.0 | 将蓝色**推离** teal |
| blue_hue_shift | +5.0 | 将蓝色**推向** cyan |
| blue_saturation_boost | 0.85 | 降饱和 -15% |
| blue_luminance_shift | -0.04 | 压暗蓝色 |
| white_balance_shift_k | +1000 | boost R, reduce B |

**发现 1: 参数方向矛盾**。`teal_push=-5.0` 和 `blue_hue_shift=5.0` 同时作用于 200–260° 范围，效果部分抵消但产生不可预测的结果。

**发现 2: 多重蓝色削弱叠加**。
- WB +1000K: B 通道 ×0.9
- blue_saturation_boost: 0.85
- 叠加效果: 纯蓝 (0,0,1) → LUT 输出 R=0.446, G=0.259, B=0.871（R 注入相当于 +45%）

**定量验证**:
| 输入 | 输出 H | ΔH | 输出 S | ΔS |
|------|--------|-----|--------|-----|
| 纯蓝 (0,0,1) | 258.3° | +18.3° | 0.702 | -30% |
| 天空蓝 (0.4,0.6,1) | 233.4° | +13.4° | 0.691 | -31% |
| 中蓝 (0,0,0.7) | 258.1° | +18.1° | 0.556 | -44% |
| 低蓝 (≤0.3) | 0° | -240° | 0 | -100% |

**低亮度蓝色完全崩塌为灰色**。B ≤ 0.3 的输入失去所有色相信息。

**建议**: 
1. 解决 teal_push / blue_hue_shift 方向矛盾 — 删除其中一个
2. 考虑 blue_saturation_boost 从 0.85 调整到 0.90–0.95
3. 评估 +1000K WB shift 的必要性（已在 split-tone 中提供暖调）

### 2.4 绿色渲染 — 结构性缺失

**发现**: LUT 中**没有专用的绿色调整参数**。

| 测量 | 纯绿 (0,1,0) | 中绿 (0,0.7,0) | 叶绿 (0.2,0.5,0.1) |
|------|-------------|---------------|-------------------|
| ΔH | +0.1° | +0.4° | -1.0° |
| ΔS | 0 | 0 | -3% |

Green 仅受到 global_saturation 和 WB shift 的间接轻微影响。

**预设描述**: "Greens lean yellow-green — warm, golden foliage"
**实际 LUT**: 绿色几乎不变

**建议**: 添加 `green_hue_shift` 和 `green_saturation_adjust` 参数（参考蓝色通道的 blue_* 参数设计）。

### 2.5 黄色渲染

**定量分析**: 纯黄 (1,1,0) → H 60→54.7° (-5.3°), R=1.0, G=0.952, B=0.460

B 通道被大幅压制（-54%），G 轻微降低（-5%），导致黄略微向绿偏移。

**预设描述**: "Yellows pop — the Gold in the name"
**实际 LUT**: 黄色保持亮度但色相 -5° 偏向绿色，与「黄金暖黄」方向略有偏差。

**建议**: 黄色在 LUT 中没有受到负面影响（亮度保持），但色相偏移可进一步微调。

### 2.6 肤色渲染

**发现**: 肤色范围上限 40° 过宽。

| 参数 | 当前值 | 5219 参考值 | 分析 |
|------|--------|------------|------|
| skin_hue_max | 40° | 35° | 40° 已入橙色区，非传统肤色 |
| skin_sat_adjust | 1.10 | 0.88 | +10% 饱和度叠加 orange_push → 过度饱和 |
| orange_push | 10° | 2° | Gold 200 可以比 5219 更暖，但 10° 可能过大 |

**定量验证**: 肤色 (0.9,0.7,0.5) → H +9.1°, S +24%

**建议**: skin_hue_max 降至 35°，skin_sat_adjust 降至 0.95–1.00。

### 2.7 三重暖调叠加

当前 LUT 通过**三条独立路径**施加暖调：
1. **per_channel_contrast** [1.02, 1.00, 0.96] — R 对比高于 B
2. **split-tone** — shadow [1.03,1.01,0.95], highlight [1.08,1.03,0.88]
3. **white_balance_shift_k** +1000K — 全局 R↑, B↓

**效果叠加**: 中灰 R/B 比 +8%，高光 +15%。这可能导致难以精确控制的整体偏暖。

**建议**: 考虑从中移除一条暖调机制（例如降低或移除 WB shift），用留存的 split-tone 精确控制暖调。

## 3. 流水线结构性限制

| 限制 | 说明 | 影响程度 |
|------|------|----------|
| 无负片反转 | 跳过 negative→positive 的物理过程 | 中 — 主要影响高光/阴影的非线性行为 |
| 无层间串扰 | 无法模拟三层乳剂的交叉光谱敏感性 | 中 — 影响色彩分离的「有机感」 |
| Rec.709 色域 | Gold 200 的某些色彩可能超出 Rec.709 | 低 — 对消费级输出影响有限 |
| 33³ 精度 | 对于强色相偏移，trilinear interpolation 可能有误 | 低 — 33³ 对此类变换通常足够 |
| **无绿色参数** | 引擎有 blue 专项但无 green 专项 | 高 — 功能缺口 |

---

## 4. 差异总结

| # | 差异 | 严重程度 | 类型 |
|---|------|----------|------|
| 1 | per_channel_contrast B 通道最低，与胶片物理相反 | 中 | 参数方向错误 |
| 2 | orange_push 对纯红色无效（hue wraparound） | 高 | Bug |
| 3 | teal_push 和 blue_hue_shift 方向矛盾 | 中 | 参数冲突 |
| 4 | 蓝色多重削弱叠加（+1000K + sat×0.85 + lum-0.04） | 中 | 参数过度 |
| 5 | 绿色渲染缺失 — 无专用参数 | 高 | 功能缺口 |
| 6 | 三重暖调叠加可能过度 | 低 | 设计考量 |

---

## 5. 建议优先级

| 优先级 | 建议 | 效果 |
|--------|------|------|
| **P0** | 修复 orange_push hue wraparound | 使红色→橙偏移对纯红色生效 |
| **P0** | 添加 green_hue_shift / green_sat 参数 | 填补绿色渲染缺口 |
| **P1** | 调整 per_channel_contrast (B ≥ 1.00) | 符合胶片物理 |
| **P1** | 解决 teal_push/blue_hue_shift 矛盾 | 统一蓝色渲染逻辑 |
| **P2** | 降低 skin_hue_max 到 35°, sat 到 1.00 | 更自然的肤色 |
| **P2** | 评估移除或降低 WB shift | 减少暖调叠加 |
