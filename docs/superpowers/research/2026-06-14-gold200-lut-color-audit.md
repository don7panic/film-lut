# Kodak Gold 200 LUT 色彩准确性审计

**日期:** 2026-06-14
**状态:** 研究报告
**研究问题:** 当前 gold200 LUT 与真实 Kodak Gold 200 胶片在色彩呈现上是否存在差异？

---

## 执行摘要

**结论：存在显著差异，特别是蓝色渲染方向完全错误。**

当前 gold200 LUT 在 8 个色彩维度中，5 个维度存在可量化的差异。最严重的问题是蓝色色相偏移方向与 Gold 200 实际特征相反（向紫色偏移 vs 应向青色偏移）。其他差异包括过度暖调叠加、肤色范围过宽、影调曲线中性灰冷偏移等。

---

## 1. 方法论

### 1.1 数据来源
- **计算分析**: 对 LUT 管线进行端到端的 test vector pass，量化色相/饱和度/亮度变化
- **社区来源**: 中文胶片社区（知乎、B站、微博、豆瓣），Kodak 官方店铺描述
- **对比基准**: Kodak Gold 200 已知特征（来自社区共识和官方材料）

### 1.2 局限
- 无法获取 Kodak E-7022 技术数据表（光谱染料密度曲线）
- 无 Gold 200 + 标准色卡的 colorimetric 测量数据
- 绿色表现数据仅来自间接推断
- Fuji C200 = Gold 200 特征曲线等价性未经独立验证

### 1.3 管线参考
```
V-Log LUT: V-Log → scene-linear → V-Gamut→Rec.709 matrix → tone curve → color grade → diplay gamma
Standard LUT: Rec.709 → inverse gamma → linear → tone curve → color grade → display gamma
Grid: 33³, trilinear interpolation, .cube format
```

## 2. 维度分析

### 2.1 🔴 红色/暖色表现

**Gold 200 实际特征:**
- 整体强烈的暖调/金色偏色（社区共识）
- 知乎用户描述："无所不在的黄调"，"像得了黄疸证" [[1]](#ref-1)
- 微博评测："一不留神就容易把人拍黄" [[2]](#ref-2)
- 淘宝评论："超级喜欢柯达的这种暖调" [[3]](#ref-3)
- 红色→琥珀偏移是 Gold 200 的标志性特征（预设文档描述）

**当前 LUT 行为:**

| 参数 | 值 | 效果 |
|------|-----|------|
| `orange_push` | 10.0 | 将 0-40° 范围色相 +10° |
| `white_balance_shift_k` | +1000K | R ×1.10, B ×0.90 |
| `highlight_tint` | [1.08, 1.03, 0.88] | 高光 R+8%, B-12% |
| `shadow_tint` | [1.03, 1.01, 0.95] | 阴影 R+3%, B-5% |

**实测色相偏移（饱和中调颜色）:**

| 输入色相 | 输出色相 | 偏移 |
|---------|---------|------|
| 0° (纯红) | 360° | **0°** ⚠️ |
| 30° (橙红) | 41° | +11° |
| 60° (黄) | 57° | -3° |

**差异分析:**
- `orange_push=10°` 对**饱和纯红色无效**（0° 偏移）。仅对中饱和/暗红有效。
- 这是 HSL 空间的固有限制：纯红色在色调旋转后，RGB 反变换会产生抵消效应。
- 三重暖调叠加（WB + tint + orange_push）可能过度。社区反馈 Gold 200 已经很强暖调，无需额外 +1000K。

**严重度: 🟡 中低** — 暖调方向正确，但机制有冗余，对纯红色效果有限。

---

### 2.2 🔵 蓝色表现 — ⚠️ 关键发现

**Gold 200 实际特征:**
- 预设描述: "Blues are warm, muted, slightly cyan"
- B站评测: Kodak Gold 200 "对天空表现力不好" [[4]](#ref-4)
- 蓝色应呈现暖调、低饱和、微偏青的复古天空感

**当前 LUT 参数:**

| 参数 | 值 | 意图 |
|------|-----|------|
| `teal_push` | **-5.0** | 推离 teal（意图让蓝色更暖） |
| `blue_hue_shift` | +5.0 | 向青色偏移 |
| `blue_saturation_boost` | 0.85 | 降低蓝色饱和度 |
| `blue_luminance_shift` | -0.04 | 蓝色更深 |

**实测蓝色色相偏移:**

| 输入色相 | 输出色相 | 偏移 | 方向 |
|---------|---------|------|------|
| 180° (青) | 177° | -3° | →绿 |
| 195° (青蓝) | 203° | **+8°** | →紫 |
| 210° (蓝青) | 224° | **+14°** | →紫 |
| 225° (蓝) | 247° | **+22°** | →紫 |
| 240° (真蓝) | 257° | **+17°** | →紫 |
| 255° (蓝紫) | 266° | +11° | →紫 |
| 270° (紫) | 274° | +4° | →紫 |

蓝色饱和度全面降至 **0.48**（从 0.70 输入）。

**🔴 严重差异：蓝色被推向紫色方向，而非青色方向！**

**根因分析:**

```
teal_push = -5.0:
  当 h < 200°: h = h + (-5) × (+1) = h - 5  → 向绿/暖
  当 h > 200°: h = h + (-5) × (-1) = h + 5  → 向紫
  
blue_hue_shift = +5.0:
  所有 200-260°: h = h + 5                   → 向紫
```

两个参数在 200-260° 范围**同向叠加**，产生 +10° 到 +22° 的紫色偏移。

- `teal_push=-5.0` 的意图是"让蓝色更暖"，但在 HSL 空间中，将蓝色推离 teal = 推向紫色 = 更暖的方向。这个效果与 `blue_hue_shift=+5.0` 叠加，产生了远超预期的紫色偏移。
- 对于 195° 附近（青蓝），两个参数**恰好抵消**（teal_push: -5 → 190°, blue_hue_shift: +5 → 195°），产生不自然的跳跃。

**严重度: 🔴 高** — 蓝色渲染方向与实际胶片相反，需要立即修正。

### 2.3 🟢 绿色表现

**Gold 200 实际特征:**
- 预设描述: "Greens lean yellow-green — warm, golden foliage"
- 没有独立社区来源专门评论 Gold 200 的绿色表现
- 基于整体暖调特性，绿色应向黄绿方向偏移是合理的推导

**当前 LUT:**
- 无专门的绿色控制参数
- 通过整体暖调（WB + tint）间接影响绿色

**实测:**
- 输入 hue ~110°（绿色 foliage test）→ 输出 hue 96°（黄绿方向）
- 偏移方向正确（-14° 向黄绿），但这是整体暖调的副作用而非有意控制

**严重度: 🟢 低** — 方向正确，缺乏精细控制。

---

### 2.4 🟡 黄色/金色表现

**Gold 200 实际特征:**
- "Gold" 在名称中，"Bright, colorful prints"（Kodak 官方描述）
- 标志性的温暖金色调

**当前 LUT:**
- `highlight_tint [1.08, 1.03, 0.88]` 在暖调高光中突出金黄
- `white_balance_shift_k=1000K` 增强全局暖金
- 实测纯黄 (1,1,0) → (1.000, 0.952, 0.460)，B 通道适度降低

**严重度: 🟢 低** — 黄色/金色表现与预期一致。

---

### 2.5 🧑 肤色渲染

**Gold 200 实际特征:**
- 预设描述: "Golden skin tones — warm, flattering, sun-kissed"
- 社区警告: "一不留神就容易把人拍黄" [[2]](#ref-2)

**当前 LUT:**

| 参数 | 值 | 分析 |
|------|-----|------|
| `skin_hue_min` | 12.0 | 合理（覆盖浅肤色） |
| `skin_hue_max` | **40.0** | ⚠️ 已进入橙色范围 |
| `skin_sat_adjust` | **1.10** | ⚠️ 增加饱和度强调暖色 |

- 典型白种人肤色: 15-35°
- 典型亚洲人肤色: 20-40°
- 40° 已接近纯橙色，加上 `skin_sat_adjust=1.10`，肤色会偏向过度暖橙

**实测:**
- Skin test (1, 0.6, 0.4) → hue 32°, sat 1.00 — 在橙红边缘
- 加上 +1000K WB 的整体暖调，最终肤色可能在 35-40° 范围

**严重度: 🟡 中低** — skin_hue_max 应降至 35° 左右，skin_sat_adjust 降至 ~1.00。

---

### 2.6 📈 影调曲线

**Gold 200 实际特征:**
- 消费级胶片，对比度高于 Portra（"consumer pop"）
- Fujicolor 200 (2021) 与 Gold 200 特征曲线相似 [[5]](#ref-5)
- 彩色负片物理结构：蓝层在最上层，通常对比度最高

**当前 LUT 参数:**

| 参数 | 值 |
|------|-----|
| `contrast` | 1.02 |
| `per_channel_contrast` | [1.02, 1.00, 0.96] |
| `black_lift` | 0.0020 |

**实测中性灰阶梯（Standard LUT 路径，tone curve only）:**

| 输入 | R | G | B | 分析 |
|------|---|---|---|------|
| 0.10 | 0.0911 | 0.0955 | **0.1049** | B 最亮 → 冷偏 |
| 0.50 | 0.4862 | 0.4931 | **0.5073** | B 最亮 → 冷偏 |
| 0.70 | 0.6900 | 0.6950 | **0.7052** | B 最亮 → 冷偏 |

**⚠️ 出人意料的发现:** `per_channel_contrast [1.02, 1.00, 0.96]` 使中性灰**向冷偏移**（B 通道最亮）。

原因: power function `x^p` 中 p < 1 时提升中灰（B: 0.96 < 1 → 提亮），p > 1 时压低中灰（R: 1.02 > 1 → 压暗）。

这与两个事实相矛盾:
1. Gold 200 是**暖调**胶片，中性灰应偏暖而非偏冷
2. 彩色负片蓝层通常对比度**最高**（物理上层），但当前 B 通道对比度最低 (0.96)

**严重度: 🟡 中低** — 冷偏移被后续 color grade 暖调抵消，但增加了 color grade 的负担。应将 per_channel_contrast 调整为 [1.02, 1.00, 1.02] 或 [1.04, 1.02, 1.06]（模拟真实三层乳剂）。

### 2.7 🎨 分色/白平衡

**Gold 200 实际特征:**
- 日光型胶片，自然光下已呈现暖调
- 社区描述为"强烈暖调"

**当前 LUT:**
- `white_balance_shift_k=1000K`: R×1.10, B×0.90
- 相当于将 5500K（日光）变为 6500K（多云/阴天）
- 与 warm split-tone + orange_push 三重叠加

**问题:**
Gold 200 本身在日光下已呈现暖调。我们的 LUT 在此基础上**再加** +1000K WB shift，导致暖调被重复计算。

**严重度: 🟡 中低** — 建议去重：保留 split-tone 暖调，降低 WB shift 至 +300-500K 或移除。

---

### 2.8 🌈 饱和度

**Gold 200 实际特征:**
- Kodak 官方: "Saturated colors"
- 预设描述: "Higher contrast than Portra — consumer film pop"

**当前 LUT:**
- `global_saturation: 1.02` — 微增
- `highlight_desat_max: 0.04` — 几乎不去饱和

**严重度: 🟢 低** — 与 Gold 200 "饱和色彩"特征一致。

## 3. 管线结构性限制

### 3.1 缺失负片反转过程
真实彩色负片: 场景光 → 负片潜影 → C-41 显影 → 橙色色罩 → 扫描去色罩+反转。

我们的 LUT 直接做正向调色，跳过了以下步骤:
- **橙色色罩 (orange mask)**: 负片基底的橙色用于补偿染料的不完美吸收。反转后的色彩平衡受此影响。
- **非线性反转**: 负片的密度-曝光曲线在反转时产生非线性映射。
- **染料云 (dye cloud)**: 胶片颗粒/染料云结构产生微观色彩混合。

3D LUT 无法模拟这些效应（特别是染料云），但通过适当的 tone curve 和 color grade 可以近似。

### 3.2 无层间交叉效应
真实彩色负片的三层乳剂（R/G/B）存在光谱交叉敏感性。当前 LUT 对每个像素独立处理，没有层间色彩混合。

### 3.3 Rec.709 色域限制
Gold 200 能记录的暖黄色/金色可能超出 Rec.709 色域。33³ LUT 在色域边界可能产生剪裁。

### 3.4 33³ 插值精度
对于 10° 级别的色相偏移，33³ 的 trilinear interpolation 精度在色域边界区域可能不足，但整体可接受。

## 4. 参数修正建议

### 🔴 高优先级 — 蓝色色相方向修复

```python
# presets/gold200.py 修改
'color': {
    # 移除或反转 teal_push（当前 -5.0 将蓝色推向紫色）
    'teal_push': 5.0,          # 改为正值: 将蓝色推**向** teal/青色
    # 或
    'teal_push': 0.0,          # 或完全移除

    # 降低 blue_hue_shift（当前 +5.0 推向紫色）
    'blue_hue_shift': -3.0,    # 改为负值: 真蓝色→青色方向
    # 注意：负值也会影响青色调（195°→更绿），需要权衡
}
```

**推荐方案:**
- `teal_push: 5.0`（正值，推向 teal）+ `blue_hue_shift: -3.0`（微向青）
- 这会使: 210° (蓝青) → ~206° (向青/teal), 240° (真蓝) → ~238° (微向青)
- 与 Gold 200 "slightly cyan" 描述一致

### 🟡 中优先级

```python
# 肤色范围收窄
'skin_hue_max': 35.0,          # 从 40.0 降至 35.0（不要进入纯橙）
'skin_sat_adjust': 1.00,      # 从 1.10 降至 1.00（不再额外增强暖色）

# 去重暖调机制
'white_balance_shift_k': 500.0,   # 从 1000 降至 500（避免与 split-tone 重复）

# 影调曲线改为暖中灰
'per_channel_contrast': [1.02, 1.00, 1.02],  # B 通道对比度提升（暖化中灰）
```

### 🟢 低优先级

```python
# 可选微调
'orange_push': 8.0,            # 从 10.0 降至 8.0（略降低暖色偏移强度）
```

## 5. 差异汇总

| # | 维度 | 差异描述 | 严重度 | 方向 |
|---|------|---------|--------|------|
| 1 | 蓝色色相 | 推向紫色 (+14~22°)，应为微青 | 🔴 高 | 完全相反 |
| 2 | teal_push + blue_hue_shift | 两个参数互相干扰/抵消 | 🔴 高 | 设计缺陷 |
| 3 | 暖调机制 | 三重叠加 (WB+tint+push) 可能过度 | 🟡 中 | 过强 |
| 4 | 肤色范围 | skin_hue_max=40° 进入橙色区 | 🟡 中 | 过暖 |
| 5 | 中灰影调 | per_channel_contrast 产生冷偏移 | 🟡 中 | 与暖调矛盾 |
| 6 | 纯红偏移 | orange_push 对纯红色无效 | 🟡 低 | 机制限制 |
| 7 | 绿色控制 | 无独立绿色参数 | 🟢 低 | 缺失 |
| 8 | 黄色/金色 | 整体正确 | 🟢 无 | — |
| 9 | 饱和度 | 整体正确 | 🟢 无 | — |
