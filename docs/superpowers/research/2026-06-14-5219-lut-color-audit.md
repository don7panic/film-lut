# Kodak 5219 LUT 色彩准确性审计

**日期:** 2026-06-14
**状态:** 最终报告（Round 1 + Round 2）
**研究问题:** 当前 5219 LUT 与真实 Kodak Vision3 5219 (500T) 胶片在色彩呈现上是否存在差异？

---

## 执行摘要

**结论：无方向性错误。5219 LUT 在三个预设中色彩准确度最高。**

与 Gold 200 LUT 的审计形成鲜明对比（蓝色色相完全反向推向紫色），5219 LUT 在全部 8 个色彩维度上**方向均正确**。唯一的中等差异是 per-channel 对比度排序可能与真实 Vision3 三层乳剂不完全一致（当前 R 通道排位偏高）。所有修正均为可选微调级别，无需强制修改。

## 1. 方法论

### 1.1 研究方法
本次审计采用**色彩逆向工程**方法：

1. **计算分析** — 端到端 test vector pass，量化色相/饱和度/亮度变化（Standard LUT 路径优先，V-Log 路径验证）
2. **参数敏感性分析** — 5 种 per_channel_contrast 方案对比
3. **网络研究** — EMULSIVE、Deakins 论坛、中文胶片社区、Film Photography Project Store
4. **第一原理推导** — 物理负片 gamma 与数字 power function 的映射关系、三层乳剂对比度机制
5. **对比审计** — 与 Gold 200 LUT 审计对照，识别共性问题

### 1.2 数据来源
- **计算验证**: test vector pass（12 色相 × 3 亮度层 + 19 点蓝色精细采样 + 4 肤色采样 + 11 步灰度阶梯）
- **社区来源**: EMULSIVE (Phil Harrison, 2017)、豆瓣胶片小组、Film Photography Project Store
- **行业论坛**: Roger Deakins 论坛 Vision3 讨论（gabj3 关于 B/G/R 曲线陡度）
- **内部文档**: `[presets/5219.py](presets/5219.py)`、`[docs/superpowers/specs/2026-06-13-5219-lut-calibration.md](docs/superpowers/specs/2026-06-13-5219-lut-calibration.md)`

### 1.3 局限
- **无法获取 Kodak H-1-5219 官方技术数据**（Kodak 官网已移除电影胶片页面，所有镜像不可用）
- **无 5219 专属 ACES IDT**（连 Vision3 200T/250D 也没有公开 IDT）
- **无标准色卡 + 真实 5219 的 colorimetric 测量数据**
- **社区报告均为定性描述**，无可量化色度数据
- **三层乳剂对比度排序来自社区推导 + Deakins 论坛暗示**，未经官方验证

### 1.4 管线参考
```
Standard LUT: Rec.709 → inverse gamma → linear → tone curve → color grade → display gamma
V-Log LUT:    V-Log → scene-linear → V-Gamut→Rec.709 matrix → linear → tone curve → color grade → display gamma
Grid: 33³, trilinear interpolation, .cube format
```

## 2. 维度分析

### 2.1 🔵 蓝色/青色阴影 — 🟢 方向正确

**5219 已知特征:**
- 经典青橙互补色调，阴影偏青蓝（cyan-biased shadows）
- 蓝层对比度最高 → cool shadows + warm highlights 的标志性分色
- EMULSIVE (Phil Harrison, 2017): "underexposed shadows had a purple hue" — **注意：仅限欠曝**

**当前 LUT:**

| 参数 | 值 | 效果 |
|------|-----|------|
| `teal_push` | **+3.0** | 将蓝色推向 teal（200° 方向） |
| `shadow_tint` | [1.00, 1.00, **1.03**] | 阴影区 B +3% |

**实测蓝色色相偏移（full pipeline, shadow region l=0.4, s=0.6）：**

| 输入 H | 输出 H | dH | 方向 |
|--------|--------|------|------|
| 180° (青) | 178° | -2° | 向绿 |
| 200° (teal) | 199° | -1° | 稳定 |
| 210° (蓝青) | 210° | 0° | 稳定 |
| 225° (蓝) | 223° | -2° | 向 teal |
| 240° (真蓝) | 233° | **-7°** | 向 teal ✓ |
| 250° | 242° | -8° | 向 teal |
| 260° (蓝紫) | 251° | -9° | 向 teal |
| 270° (紫) | 262° | -8° | 向 teal |

**🔑 关键结论：蓝色 → teal/青色，方向完全正确。** 与 Gold 200（`teal_push=-5.0` 将蓝色推向紫色）形成鲜明对比。

**偏移量分析：**
- 偏移由两部分组成：`teal_push=+3.0` (直接 hue shift) + per_channel B 最高 → warm bias（额外 -4°~-6°）
- 在 260° 处达最大 -9°。如果觉得偏强，可降 `teal_push` 到 2.0

**关于 "purple shadow" 的澄清 [Round 2 解决]：**
- EMULSIVE 说的是**"underexposed shadows"**（欠曝阴影）偏紫，不是正常曝光阴影
- 欠曝紫色 = 染料层形成不充分 + 橙色色罩主导 → 扫描反转后 magenta crossover
- 正常曝光 = cyan-biased（蓝层对比度最高）
- **矛盾不存在。我们的 cyan shadow 方向完全正确。**

**严重度: 🟢 低** — 方向正确，力度适中。

### 2.2 🧑 肤色渲染 — 🟢 良好

**5219 已知特征:**
- EMULSIVE: "lovely skin tones"（【已确认】）
- 暖金色肤色，高光平滑过渡

**当前 LUT:**

| `skin_hue_min` | 10.0 | 覆盖浅肤色 ✓ |
| `skin_hue_max` | 35.0 | 覆盖亚洲+白种人肤色，不进入纯橙 ✓ |
| `skin_sat_adjust` | 0.88 | 降低 12% 饱和度，柔化肤色 |

**实测肤色（full pipeline）：**

| 输入 RGB | 输入 H | 输出 H | dH | 输入 S | 输出 S |
|---------|--------|--------|------|--------|--------|
| (1.0, 0.60, 0.40) 浅肤色 | 20° | 20° | 0° | 1.00 | 0.85 |
| (0.85, 0.55, 0.35) 中浅 | 24° | 24° | 0° | 0.62 | 0.50 |
| (0.70, 0.42, 0.28) 中等 | 20° | 20° | 0° | 0.43 | 0.33 |

**分析：**
- 肤色色相几乎不变（dH ≤ 2°），保护非常好 ✓
- 饱和度适度降低，柔化效果良好 ✓
- `skin_hue_max=35°` 刚好覆盖典型肤色，不进入纯橙色 ✓

**与 Gold 200 对比：** Gold 200 的 `skin_hue_max=40°`（进入纯橙）、`skin_sat_adjust=1.10`（增加饱和度）。5219 的参数更保守、更准确。

**严重度: 🟢 低** — 肤色保护良好，参数合理。

### 2.3 📈 影调曲线 / 对比度 — 🟡 排序待验证

**5219 已知特征:**
- 中等对比度，Vision3 系列中速度最快（ISO 500）
- 优秀高光保持（14-15 档动态范围）
- 三层乳剂结构：蓝层对比度最高，红层最低（社区推导，[UNVERIFIED]）
- Deakins 论坛 (gabj3): "Blue and Green curves are slightly steeper when compared to the Red curve" → 暗示 **B≈G > R**

**当前 LUT 参数:**

| 参数 | 值 |
|------|-----|
| `contrast` | 1.08 |
| `per_channel_contrast` | [1.02, 1.00, 1.06] — R, G, B |
| `black_lift` | 0.0018 |
| `shadow_toe_pivot` | 0.12 |
| `shadow_toe_power` | 0.82 |
| `highlight_shoulder_start` | 0.72 |
| `highlight_shoulder_power` | 1.25 |

**实测中性灰阶梯（Standard 路径，tone curve only）：**

| 输入 | R | G | B | R-B 差 | 偏色 |
|------|---|---|---|--------|------|
| 0.000 | 0.02814 | 0.03173 | 0.02213 | +0.00601 | 中性 |
| 0.200 | 0.19052 | 0.19764 | 0.17688 | +0.01364 | 暖 |
| 0.500 | 0.46494 | 0.47220 | 0.45071 | +0.01423 | 暖 |
| 0.700 | 0.67477 | 0.68006 | 0.66429 | +0.01047 | 暖 |
| 1.000 | 1.00000 | 1.00000 | 1.00000 | +0.00000 | 中性 |

**per-channel 对比度分析（Round 2 敏感性测试）：**

各通道有效 power = `contrast × per_channel[c]`:

| 方案 | per_channel | R_eff | G_eff | B_eff | 排序（暗→亮） | 备注 |
|------|-------------|-------|-------|-------|---------------|------|
| **CURRENT** | [1.02,1.00,1.06] | 1.102 | 1.080 | 1.145 | B < R < G | R 排第二 |
| Option A | [0.98,1.00,1.06] | 1.058 | 1.080 | 1.145 | B < G < R | R 最低 ✓ |
| Option B | [1.00,1.04,1.06] | 1.080 | 1.123 | 1.145 | B < G < R | Deakins B≈G>R |
| **Option C** | **[1.00,1.02,1.06]** | 1.080 | 1.102 | 1.145 | B < G < R | **保守推荐** |
| Option D | [1.01,1.00,1.04] | 1.091 | 1.080 | 1.123 | B < R < G | 阿花建议 |

**分析：**
- CURRENT [1.02, 1.00, 1.06] 使 R 通道对比度排在 G 之前（B<R<G），中性灰偏暖但不完全符合 Vision3 物理排序（推测为 B>G≥R）
- Option C [1.00, 1.02, 1.06] 使排序变为 B<G<R，更符合 Deakins 数据暗示的 B≈G>R
- 差异极小：中性灰 R-B diff 从 +0.0095 变为 +0.0143（增加 0.005），肤色 dH 不变
- **注意**：Deakins 论坛原帖未能在 Round 2 定位（可能已移除），B≈G>R 的排序仅来自 agent 引用，需进一步验证

**关于 gamma ~0.55 与 power 1.08 的关系：**
- 胶片特性曲线的 gamma ~0.55 是 log(密度) vs log(曝光) 的斜率
- 我们的 power 1.08 是线性 vs 线性的 exponent
- 两者不在同一坐标空间，直接比较无意义
- power > 1 在 scene-linear 空间产生 S 形曲线（midtones 对比度增强），方向合理
- 这是"调色后"目标的合理近似，无需与物理负片 gamma 对齐

**严重度: 🟡 中** — per-channel 排序不完全确定，建议 Option C 微调。但当前值也在可接受范围内。

## 3. 管线结构性限制

### 3.1 缺失负片反转过程 🔧
真实光路：场景光 → 负片潜影 → ECN-2 显影 → 橙色色罩 → 扫描去色罩+反转。
我们的 LUT 直接做正向调色，跳过了橙色色罩和非线性反转。3D LUT 无法模拟染料云效应。

### 3.2 无层间交叉效应 🔧
真实三层乳剂存在光谱交叉敏感性。当前 LUT 对像素独立处理。

### 3.3 Rec.709 色域限制 🔧
5219 可能记录超出 Rec.709 的暖黄/金色。33³ LUT 在色域边界可能剪裁。

### 3.4 V-Log 路径采样特性 ℹ️
V-Log LUT 的 33³ 均匀采样在 V-Log 域，导致阴影分辨率为零（V-Log 码值 < 0.125 全部映射到线性 0）。这是 V-Log 编码的固有限制。

### 3.5 HSL 空间近黑色伪影 ℹ️
近黑色像素 (0,0,0) 在 RGB→HSL 转换时产生 dH=+84° 的随机色相。虽在实际使用中不可见（亮度 ~0.03），但表明管线在理论边界有不稳定点。

## 5. 参数修正建议

### 🟡 中优先级 — per_channel_contrast 微调

当前 `[1.02, 1.00, 1.06]` 使 R 通道对比度排在 G 之前（排序 B>R>G），而 Vision3 物理推测应为 B>G≥R。

**推荐（保守）：**
```python
# presets/5219.py 第17行
'per_channel_contrast': [1.00, 1.02, 1.06],  # R 降低，G 提升至接近 B
```
- 排序变为 B>G>R，更符合三层乳剂物理
- 影响极小：中性灰 R-B diff +0.0095 → +0.0143（+0.005），肤色 dH 不变
- 如需更激进匹配 Deakins 数据：`[1.00, 1.04, 1.06]`

### 🟢 低优先级 — 可选微调

```python
# 1. 如果觉得蓝色偏移偏强，可降低 teal_push
'teal_push': 2.0,  # 从 3.0 降至 2.0（减少蓝→青偏移）

# 2. 如果觉得肤色去饱和过多
'skin_sat_adjust': 0.92,  # 从 0.88 升至 0.92

# 3. 更低的全局对比度（更接近 5219 低反差特征）
'contrast': 1.05,  # 从 1.08 降至 1.05
```

**5219 标定质量远高于 Gold 200。** 这可能是两方面的原因：① 5219 有单独的研究标定文档 + v1→v2 校准过程；② 5219 的胶片特征本身更"中性"（作为电影负片，比消费级 Gold 200 更可控）。
