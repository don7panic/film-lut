# Round 2: 流水线结构性限制分析

## 1. 无反转向过程

### 现象
当前 LUT 流水线直接模拟最终正向外观：
```
V-Log → linear → Rec.709 → tone curve → color grade → display gamma → output
```

### 真实胶片过程（缺失的步骤）
```
场景光 → 曝光 → 负片潜影 → C-41 显影 → 橙色色罩 (orange mask)
→ 扫描/放大 (包含: 色罩去除 + 反转 + 色彩校正)
```

### 差异分析

| 真实胶片步骤 | 当前 LUT 中的等效 | 缺失 |
|-------------|-----------------|------|
| **负片 H&D 曲线** | 无等效。正向 S-curve 近似。 | 负片的非线性反转特性（阴影 gamma 不同于高光 gamma）无法用正向幂函数精确模拟 |
| **橙色色罩 (orange mask)** | 无等效。直接在 Rec.709 空间调色。 | 色罩的光谱选择性吸收产生独特的色彩耦合效应，特别是青/蓝色调区域的色相扭曲 |
| **扫描反转向** | tone curve 反转近似 | 扫描过程本身引入的白平衡、曝光、色彩矩阵变换被省略 |
| **化学串扰 (interlayer)** | 无等效。独立 R/G/B 通道处理。 | 真实负片三层乳剂间有 interimage effects：一层染料的形成会影响邻层的显影，产生独特的色彩饱和度增强和色相偏移 |

**严重度**: ⚠️ 中等。大多数消费者级 LUT 也跳过这些步骤。但对追求高度准确的胶片模拟来说，这是根本性限制。

**证据级别**: 高（彩色负片化学过程的公认事实）

## 2. 无层间串扰 (Interlayer Crosstalk)

真实彩色负片的 cyan/magenta/yellow 染料层并非完全独立：
- 品红染料形成时会抑制青染料的形成（或反之），产生色彩增强
- 这种 "interimage effect" 使胶片色彩饱和度高于单纯的独立通道处理
- **3D LUT 可以在一定程度上近似此效应**（通过非零的交叉通道值），但当前独立 R/G/B 参数管线无法产生此效果

**严重度**: 🟡 低-中。3D LUT 格式本身可以编码交叉通道效应，但当前**参数化管线设计**（R/G/B 独立处理 + HSL 全局调整）无法表达。

**证据级别**: 高（JOSA 1952, "Subtractive Color Reproduction: Interimage Effects"）

## 3. Rec.709 色域限制

### 矩阵分析

V-Gamut → Rec.709 转换矩阵 `M_VGAMUT_TO_REC709`：
```
[[ 1.807 -0.696 -0.111]
 [-0.170  1.306 -0.136]
 [-0.025 -0.154  1.180]]
```

**实测**: V-Gamut 纯色主色输入后，Rec.709 输出大幅超出 [0,1]：
- 红色 [1,0,0] → [1.807, 0, 0] 🔴 80% 超出
- 绿色 [0,1,0] → [0, 1.306, 0] 🔴 31% 超出
- 黄色 [1,1,0] → [1.111, 1.136, 0] 🔴 两边均超出

### 处理方式

流水线中的 `vgamut_to_rec709()` 使用 `np.clip(result, 0.0, None)` — 仅裁剪负值，**不裁剪 >1.0 的值**。超出的值在后续 tone curve 的 highlight shoulder 阶段被压缩回 [0,1]。

### 影响

Gold 200 以金黄色调著称。在 V-Gamut 中，金色/黄色区域（高 R+G，低 B）会产生超出 Rec.709 的值。这些颜色在 highlight shoulder 压缩后**失去饱和度**——这与 Gold 200 的 "bright, colorful prints" 描述矛盾。

**具体量化**: 金色 [1.0, 0.8, 0.0] (V-Gamut) → tone mapped 后 R 被大幅压缩、G 几乎归零，输出极暖但**完全失去黄色特征**的红色调。

**严重度**: 🟡 中等。对于 Gold 200 这种以金黄色为卖点的胶片，Rec.709 色域限制是一个实质性损失。

**证据级别**: 高（直接矩阵计算）

## 4. 33³ LUT 精度限制

### 实测数据

使用 1000 个随机 V-Log 颜色测试（trilinear interpolation vs 直接管线计算）：

| 指标 | 值 | 8-bit LSB 等效 |
|------|-----|---------------|
| Mean error | 0.0034 | <1 LSB |
| Median error | 0.0000 | 0 LSB（精确命中网格点） |
| P95 error | 0.0286 | ~7 LSBs 🔴 |
| P99 error | 0.1055 | ~27 LSBs 🔴 |
| Max error | 0.2240 | ~57 LSBs 🔴 |
| >1 LSB (>0.004) | 10% of pixels | — |
| >13 LSBs (>0.05) | 3% of pixels | — |

### 分析

- **中位数误差 = 0**: 大多数像素落在网格点附近，插值精确
- **P95 误差 7 LSBs**: 意味着 5% 的像素有可见的色差
- **3% 像素有严重误差 (>13 LSBs)**: 在平滑渐变中可能产生**可见的色带/色阶不连续**
- V-Log 管线的极端非线性（指数/对数转换 + 矩阵 + S-curve）使 33³ 网格在部分区域严重欠采样

### 65³ 对比

生成 65³ LUT 并测试：**精度无改善**（均值 0.0104 vs 0.0034，最大值 0.242 vs 0.224）。误差主要来源是管线的极端非线性本身，而非网格密度的简单扩展能解决。

**严重度**: 🟡 中等。33³ 对 Standard LUT（仅 creative grade）足够，对 V-Log LUT（包含 V-Log 解码+色域转换+调色）**勉强可接受但存在可见伪影风险**。

**证据级别**: 高（直接测量）

## 5. V-Log 阴影死区

### 实测

V-Log 代码值在 0-0.125 范围全部映射为 scene-linear 0.0：

| V-Log 输入 | Scene-linear 输出 |
|-----------|-------------------|
| 0.050 | 0.000000 |
| 0.100 | 0.000000 |
| 0.125 | 0.000000 |
| 0.150 | 0.004464 |
| 0.181 | 0.010000 |

### 影响

在 33³ LUT 中，R/G/B 各通道的前 ~4 个网格点（共 33）输出均为纯黑。这意味着 **V-Log LUT 丢失了约 12% 的阴影编码范围**。对于在相机内使用 V-Log LUT 拍摄的场景，暗部细节可能被不可逆地裁剪。

**严重度**: 🔴 高。这是一个功能性缺陷——V-Log 阴影信息在 LUT 中永久丢失。

**证据级别**: 高（直接计算，V-Log 逆函数的标准行为）

## 6. V-Log vs Standard LUT 路径对比

### 差异

| 维度 | V-Log LUT | Standard LUT |
|------|----------|-------------|
| 输入空间 | V-Log/V-Gamut | Rec.709 显示 |
| 编码内容 | V-Log 解码 + 色域转换 + creative grade | 逆 gamma + creative grade |
| 精度压力 | **极高**（编码三个变换在一个 3D grid） | 低 |
| 阴影死区 | 是（12% 范围丢失） | 否 |
| 色域压缩 | 是（V-Gamut→Rec.709 裁剪） | 否 |

### 建议

V-Log LUT 应拆分：
1. **1D LUT**（V-Log → scene-linear 色调映射）
2. **3×3 矩阵**（V-Gamut → Rec.709）
3. **3D LUT**（creative grade + display gamma）

但 S9 相机可能不支持链式 LUT。替代方案：增大 V-Log LUT 到 65³ + 使用 tetrahedral interpolation。

**严重度**: 🟡 中等（受限于相机硬件能力）

**证据级别**: 高（直接测量 + 架构分析）

## 7. 其他项目架构参考

### 行业标准
- **OpenColorIO**: 链式色彩管理（1D LUT → 3×3 matrix → 3D LUT）。电影/VFX 行业标准
- **ACES**: Academy Color Encoding System，标准化色彩管线

### 胶片模拟工具
- **Dehancer**: 多阶段: 输入变换 → 负片模型 → 正片/印片模型 → 输出变换 → 颗粒/halation/bloom 效果
- **FilmConvert**: 相机特定输入变换 + 胶片 stock 3D LUT + 颗粒
- **CinePrint16**: DaVinci Resolve 节点树，模拟 16mm 胶片流程

### 与当前架构的对比
| 特性 | 当前架构 | 行业最佳实践 |
|------|---------|------------|
| 输入 | 单一 LUT | 分离的输入变换（1D/3×3） |
| 负片 | 无 | 专用负片响应模型 |
| 色罩 | 无 | orange mask 模拟 |
| 调色 | 参数化 HSL | 3D LUT（从真实扫描生成） |
| 颗粒/光晕 | 无 | 物理模拟 |

**证据级别**: 高（公开文档）
