# Hasselblad HNCS 色彩差异审计 — 最终报告

**日期:** 2026-06-14
**状态:** 最终报告
**核心问题:** 当前 hassy_blue LUT 与哈苏 HNCS 真实色彩特征在哪些维度存在差异，差异程度如何？

---

## 执行摘要

**结论：存在 3 个高严重度差异和 3 个中等差异。当前 LUT 在核心设计理念上与 HNCS 存在方向性偏差，特别是影调曲线和蓝色渲染机制。**

最严重的问题是：(1) 影调曲线**降低**而非增强中灰反差，与 HNCS「感知增强」描述相反；(2) 蓝色色相存在 S 形不连续偏移，破坏了哈苏蓝应有的「干净统一」感；(3) 白平衡偏移是蓝色饱和度的主要驱动因素（贡献 80%），而非 HNCS 所描述的「整体色准的自然结果」。

---

Standard LUT: Rec.709 → inverse gamma → scene-linear → tone curve → color grade → display gamma
V-Log LUT:    V-Log → scene-linear → V-Gamut→Rec.709 matrix → tone curve → color grade → display gamma
Grid: 33³, trilinear interpolation, .cube format

### 1.3 HNCS 五项核心原则（验证基准）

| # | 原则 | 来源 |
|---|------|------|
| P1 | 「哈苏胶片曲线」— 感知增强型 tone mapping，中灰区保持丰富反差 | 哈苏官方 |

| P5 | L*RGB 色彩空间 — 在感知均匀空间中做变换 | 哈苏官方 |

## 2. 维度分析

### 2.1 🔴 D1: 影调曲线 — 当前实现与「哈苏胶片曲线」相反

**HNCS 描述:**
- 「哈苏胶片曲线」让中灰区保持**丰富反差**，暗部和高光自然过渡
- 不是技术 gamma，而是「感知增强」曲线
- "film-like image quality with true-to-life, wide-ranging contrast"

**当前 LUT 实测（tone curve only, neutral gray）:**

| 亮度区间 | 输入范围 | 局部斜率 | 反差分 |
|---------|---------|---------|--------|
| 深阴影 | 0.01–0.08 | **2.723** | 极高 |
| 阴影 | 0.12–0.23 | **1.217** | 高 |
| 中低 | 0.25–0.40 | **0.831** | 低于 1.0 |
| 中灰 | 0.40–0.60 | **0.639** | 低于 1.0 ⚠️ |
| 中高 | 0.60–0.75 | **0.482** | 低于 1.0 ⚠️ |
| 高光 | 0.85–0.98 | **0.502** | 低于 1.0 |

**有效 gamma at mid-gray (0.50): 0.421**（极低，强烈提亮中灰）

```
当前参数: contrast=0.95, shadow_toe_power=0.80, highlight_shoulder_power=1.22
```

**🔴 根本矛盾:** `contrast=0.95` (< 1.0) 使 `x^0.95 > x`（对所有 x < 1），即**提亮**而非压暗中灰。HNCS 描述的「增强中灰反差」需要 contrast **> 1.0** 来压暗中间调，配合 shadow toe **< 1.0** 补偿暗部、highlight shoulder **> 1.0** 保护高光。

当前曲线整体输出 > 输入（中灰 0.50 → 0.747），产生「漂浮」感而非「扎实」的胶片感。

**严重度: 🔴 高** — 影调方向与 HNCS 核心设计理念相反。这是「灵魂」级问题（HNCS 称影调曲线是灵魂）。

---

### 2.2 🔴 D2: 蓝色渲染 — 色相 S 形不连续 + 饱和度过度增强

**HNCS 描述:**
- 「哈苏蓝的深邃通透，不是靠单独推蓝色通道实现的」
- 蓝色是「结果」不是「手段」
- 来自整体色彩准确 + 影调自然 + 肤色不偏的综合效果
- 社区描述：干净、深邃、有层次感、3D 感蓝调

**当前 LUT 实测（蓝色色相偏移，sat=0.7, lum=0.5）:**

| 输入色相 | 输出色相 | ΔH | 偏移方向 |
|---------|---------|-----|---------|
| 180° (青) | 182.0° | +2.0° | →紫 |
| 195° (青蓝) | 191.3° | **−3.7°** | →绿 |
| 210° (蓝青) | 203.9° | **−6.1°** | →绿 |
| 225° (蓝) | 219.1° | **−5.9°** | →绿 |
| 237.5° | 237.0° | −0.5° | ≈中性 |
| 240° (真蓝) | 241.8° | +1.8° | →紫 |
| 255° (蓝紫) | 262.5° | **+7.5°** | →紫 |
| 260° | 267.7° | **+7.7°** | →紫 |
| 270° (紫) | 274.4° | +4.4° | →紫 |

**🔴 S 形不连续:** 200-238° 区间色相向**绿色**偏移（最大 −6.5°），240-270° 区间向**紫色**偏移（最大 +7.7°）。过渡点附近存在约 3°/步的突变。这种 S 形弯曲会破坏天空渐变中的蓝色统一感，产生不自然的色彩断裂。

**蓝色饱和度分解（h=240°, sat=0.7）:**

| 管线步骤 | 输出饱和度 | 贡献 |
|---------|-----------|------|
| 完整管线 | **0.808** | — |
| 无 WB 偏移 | 0.727 | Δ=−0.081 (80%) |
| 无 blue_boost | 0.778 | Δ=−0.029 (20%) |

```
blue_saturation_boost=1.05 的意图: 5% 增幅
WB shift=-400K 的实际贡献: +8.1% 增幅（B×1.04, R×0.96）
蓝色净饱和度变化: +15.4%（远超预期）
```

**🔴 双重矛盾:**
1. 白平衡偏移贡献了蓝色饱和度增强的 **80%**，而 HNCS 明确说蓝色不靠独立通道增强
2. 蓝色饱和度实际增幅（+15.4%）远超参数意图（+5%），低饱和蓝色增幅更夸张（+78% at sat=0.1）

**严重度: 🔴 高** — 方向性违背 HNCS 核心原则 P4，S 形不连续破坏色彩统一感。

### 2.3 🟡 D3: 肤色还原 — 范围偏宽，保护机制简单

**HNCS 描述:**
- 「uncompromising skin tone reproduction」
- 即使拉曲线、调对比度，肤色也不能崩
- 不是简单的 hue 遮罩，而是深度优化

**当前 LUT 实测:**

| 输入色相 | 输出饱和度 (sat=0.5, lum=0.5) | ΔSat | 是否在保护范围内 |
|---------|---------------------------|------|----------------|
| 10° | 0.456 | −0.044 | ✓ (8-42°) |
| 15° | 0.456 | −0.044 | ✓ |
| 20° | 0.456 | −0.044 | ✓ |
| 25° | 0.456 | −0.044 | ✓ |
| 30° | 0.456 | −0.044 | ✓ |
| 35° | 0.456 | −0.044 | ✓ |
| 40° | 0.493 | −0.007 | ✓ |
| 42° | 0.493 | −0.007 | ✓ |
| 45° | 0.491 | −0.009 | — (超出范围) |

```
皮肤色相科学参考:
  白种人: 15°–35°
  亚洲人: 20°–40°
  非洲裔: 15°–35°
当前范围: 8°–42°
```

**分析:**
- `skin_hue_max=42°` 已进入纯橙色区（40°+），需收紧至 38-40°
- 保护机制是简单的 `s × 0.92` 乘法，缺乏基于 L*RGB 的感知保护
- 皮肤范围内部效果不均：10-35° 获得 −8.8% 减饱和，40-42° 仅 −1.4%
- HNCS 描述的是「即使在曲线调整下也不崩」，但我们只能通过固定参数保护

**严重度: 🟡 中** — 方向正确，范围偏宽、机制简单。

---

### 2.4 🟡 D4: 整体饱和度 — 分布不均，偏差方向合理

**HNCS 描述:**
- 「自然但优于真实」
- 色彩自然但不平淡，反差真实但不硬

**当前 LUT 实测（sat=0.7, lum=0.5）:**

| 颜色 | ΔS | 方向 |
|------|-----|------|
| 红色 | **−0.020** | 降低 |
| 橙色 | **−0.070** | 大幅降低 |
| 黄色 | −0.006 | 微降 |
| 绿色 | +0.023 | 微增 |
| 青色 | +0.059 | 增加 |
| 蓝色 | **+0.108** | 大幅增加 |
| 紫色 | +0.073 | 增加 |
| 品红 | −0.020 | 降低 |

**7 色调平均: +0.024**（净增饱和度）

**分析:**
- 饱和度分布呈现清晰的「冷增强/暖降低」模式，这是 WB=−400K 的直接后果
- HNCS 应该是整体自然、均匀的饱和度，而非选择性地偏冷
- `global_saturation=0.94` 的降饱和意图被 WB 偏移部分抵消

**严重度: 🟡 中** — 色彩平衡偏离中性，蓝色过度突出。

---

### 2.5 🟡 D5: 白平衡 — 冷偏度过强

**当前 LUT:**
```
white_balance_shift_k: −400K
R factor: 0.960 (−4%)
B factor: 1.040 (+4%)
```

**对中灰的累计影响:**

| 输入 | 输出 R | 输出 G | 输出 B | B−R 偏差 |
|------|--------|--------|--------|---------|
| 0.18 (中灰) | 0.4482 | 0.4563 | 0.4664 | **+0.0181** |
| 0.50 | 0.7197 | 0.7331 | 0.7476 | **+0.0279** |
| 0.85 | 0.8973 | 0.9139 | 0.9319 | **+0.0346** |

**分析:**
- B−R 偏差随着亮度增加而增大（0.018 → 0.028 → 0.035），高光比暗部更偏冷
- −400K 是中等冷偏（大约相当于日光→阴影的色温降幅）
- HNCS 的「clean cool」应该是**微妙的**，−200K 可能更合适
- 更重要的是，这个冷偏是造成 D2/D4 问题的根本原因

**严重度: 🟡 中** — 配合 tint 叠加，整体偏冷可能过强。

---

### 2.6 🟢 D6: 阴影/高光过渡

**阴影:**
```
shadow_tint: [1.00, 1.00, 1.01] — B+1%，极轻微冷阴影
实测暗部 (0.05): R=0.2335, G=0.2381, B=0.2442 → 冷偏
```

**高光:**
```
highlight_tint: [1.00, 1.00, 1.00] — 中性高光
实测高光 (0.85): R=0.8973, G=0.9139, B=0.9319 → 冷偏（来自 WB）
```

**影调曲线特征:**
- `black_lift=0.00120`: 8-bit 层面 = 0（实际上没有胶片基底雾效果）
- `shadow_toe_power=0.80`: 强烈提亮暗部（配合 contrast=0.95 加剧了整体提亮）
- `highlight_shoulder_start=0.72, power=1.22`: 高光 roll-off 偏早启动，压缩温和

**严重度: 🟢 低** — tint 参数保守合理，但被 WB 冷偏覆盖。black_lift 过小，缺乏胶片感。

### 2.7 🔴 D7: 色彩空间 — Rec.709/HSL vs L*RGB 的结构性差异

**HNCS 实际使用:**
- L*RGB — 哈苏自研的 LAB 派生色彩空间，感知均匀
- 在 L*RGB 中做变换，保证色彩过渡在各方向上都平滑均匀
- 配合 Phocus 的 L*RGB 工作空间，编辑时色彩损失最小

**我们的实现:**
- Rec.709 线性 RGB + HSL 色彩空间
- HSL 是**非感知均匀**的：等步长的 ΔH 在不同色相/饱和度下产生的视觉差异不同

**具体影响:**

| 操作 | 在 L*RGB 中 | 在 HSL 中 |
|------|------------|----------|
| 色相偏移 | 在蓝色和黄色区域等幅度偏移，感知效果不同 | 等 ΔH 步长，但在蓝色区 (低亮度) 和黄色区 (高亮度) 的感知色差不同 |
| 饱和度调整 | 等量调整在不同色相感知均匀 | 等量 ΔS 在蓝/黄色区的视觉冲击力不同 |
| 亮度调整 | 基于 CIELAB L*，匹配人眼亮度感知 | 基于平均 RGB 值，不等同于人眼感知 |

**严重度: 🔴 高**（结构性限制）— 无法通过参数修复。但对于 Rec.709 显示场景，影响有限。

### 2.8 🟢 D8: 其他管线结构性限制

| 限制 | HNCS | 我们的实现 | 影响 | 可修复？ |
|------|------|-----------|------|---------|
| 逐像素标定 | ✅ 每台相机逐像素标定 | ❌ 通用 33³ LUT | 基础色温/DR 精度差异 | ❌ 硬件限制 |
| 光谱敏感度 | 已知传感器光谱响应 | V-Log 近似 | 同色异谱误差 | ❌ |
| 16-bit 处理 | 16-bit 色彩深度 | float64（足够） | — | ✅ |
| 场景参照 | 已知场景光照 → 输出 | 线性 RGB pipeline | 白平衡自动适配 | ❌ |

**严重度: 🟢 低/信息** — 这些是无法逾越的硬件/系统差异。

## 4. 参数修正建议

### 🔴 高优先级

#### 4.1 影调曲线重做（D1）

```python
# presets/hassy_blue.py 修改
'tone': {
    'black_lift':                0.0040,       # ↑ 从 0.0012 提升（可见的胶片基底）
    'shadow_toe_pivot':          0.12,          # ↑ 从 0.10（给 toe 更多空间）
    'shadow_toe_power':          0.85,          # ↑ 从 0.80（减弱暗部提亮）
    'contrast':                  1.08,          # ← 关键！从 0.95 改为 >1.0（压暗中灰，增强反差）
    'per_channel_contrast':      [1.00, 1.00, 1.00],  # 保持色度中性
    'highlight_shoulder_start':  0.68,          # ↓ 从 0.72（早启动 shoulder 补偿 contrast 提升）
    'highlight_shoulder_power':  1.35,          # ↑ 从 1.22（更强的 shoulder 保护高光细节）
},
```

**设计原理:**
- `contrast=1.08`: x^1.08 < x（对 x<1），压暗中灰 → 增强中灰区反差 → 匹配 HNCS P1
- `shadow_toe_power=0.85`: 仍保持 <1.0 提亮暗部，但减弱幅度，避免与 contrast>1 形成「两头翘」
- `highlight_shoulder_power=1.35`: 更强的 shoulder 压缩，保护高光 roll-off
- `black_lift=0.0040`: 提升到可见水平（8-bit: ~1），模拟胶片片基

#### 4.2 蓝色渲染修正（D2）

```python
# presets/hassy_blue.py 修改
'color': {
    # 降温白平衡 — 这是 S 形不连续和饱和度过高的根源
    'white_balance_shift_k': -150.0,      # ← 从 -400 减至 -150（微妙清冷）
    
    # 蓝色参数调整为真正的「微调」而非主驱动
    'blue_saturation_boost':   1.03,       # ← 从 1.05 降至 1.03
    'blue_luminance_shift':   -0.01,       # ← 从 -0.02 减半（深蓝但不过度）
    'blue_hue_shift':          0.0,        # ← 从 +1.5 改为 0（移除 hue shift 避免 S 形）
},
```

**设计原理:**
- `white_balance_shift_k=-150K`: 微妙清冷（R×0.985, B×1.015），让蓝色从整体色准中浮现，而非通过渠道增幅制造
- `blue_hue_shift=0`: 移除会导致 S 形不连续的 hue shift
- 蓝色饱和度: WB贡献(≈3%) + blue_boost(3%) ≈ 6% 增幅，足够产生「哈苏蓝」的深邃感

### 🟡 中优先级

#### 4.3 肤色范围收窄（D3）

```python
'skin_hue_min': 10.0,          # ↑ 从 8.0（收紧到典型肤色起点）
'skin_hue_max': 38.0,          # ↓ 从 42.0（不进入纯橙区）
'skin_sat_adjust': 0.94,       # ↑ 从 0.92（稍微减弱减饱和幅度）
```

### 🟢 低优先级（可选）

```python
# 使全局饱和度更均匀
'global_saturation': 0.96,         # ↑ 从 0.94（WB降温后需要补偿）

# 阴影微暖化（抵消 WB 冷偏）
'shadow_tint': [1.005, 1.00, 1.00],   # 暗部 R+0.5%，微暖平衡

# 高光保持中性
'highlight_tint': [1.00, 1.00, 1.00],  # 不变
```

## 5. 管线结构改进建议

### 5.1 将 WB 操作移到 HSL 转换之后
当前 WB 在 RGB 阶段应用（Step 2），导致 B channel boost 渗透到所有后续 HSL 操作。建议将 WB 移到 HSL 空间，直接操作色温分量（hue shift），避免 channel-level 交互。

### 5.2 LUT 尺寸升级
当前 33³ 对于 10° 级别的色相偏移足够，但考虑在色域边界区域可能有精度损耗。建议对交付版使用 65³。

### 5.3 L*RGB 近似
无法实现真正的 L*RGB，但可以在 HSL 操作中引入 CIELAB L* 作为亮度参考（替代 luma），使亮度调整更贴合人眼感知。

1. **需要实际哈苏样张比对**: 使用 Hasselblad X1D/X2D 的 JPEG 直出 + ColorChecker 获取实际色度数据，才能做真正的地面真值对比
2. **V-Log 路径验证**: 当前 V-Log→linear 解码在低值区可能有问题（R2E 发现灰色输出为 0），需要独立验证
3. **修正版 A/B 测试**: 生成修正参数版 LUT 与当前版做视觉并排对比
4. **L*RGB 可行性评估**: 评估在 Python pipeline 中引入 CIELAB 转换的复杂度和性能影响
5. **逐像素标定的近似**: 研究是否可以通过 noise pattern / dithering 模拟部分逐像素标定效果
